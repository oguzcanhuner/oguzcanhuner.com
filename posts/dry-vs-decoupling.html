In the begining things were DRY

DRY, also known as Don't Repeat Yourself, is a pretty basic programming principle which most programmers learn early on in their careers. It's very easy to pick up; if you have two or more bits of code that do the same thing, you should define that code in one place where it can be shared. This means that if you need to change the behaviour of the code, there's now only one place that you need to change it in.

Coupling

Software coupling is essentially a measure of how much bits of code in a system rely on each other. Each individual case of code relying on other code is called software dependancy.

An object depends on another object if, when one object changes, the other might be forced to change in turn
